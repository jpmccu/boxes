Building an OWL Ontology
with Boxes

Model a class hierarchy, add individuals, define object properties, and export to Turtle — all in the browser, no install required.

What you'll build

A simple Animals ontology in OWL 2, built from scratch. By the end you will have:

  • An ontology header node (owl:Ontology)
  • A class hierarchy — LivingThing → Animal → Mammal/Bird → Dog/Cat/Eagle
  • Named individuals (Rex, Fido, Tweety) typed via rdf:type
  • An object property (ex:hasOwner) linking animals to people
  • A valid Turtle / RDF export ready for Protégé or any SPARQL endpoint

The interactive panels below are live Boxes editor instances — you can zoom, pan, click nodes, and explore each stage of the ontology as you follow along.

1

Start from the template

Open Boxes and click "Ontology or RDF File" on the welcome screen. This starts you with the OWL palette and stylesheet pre-configured — node types, edge types, and Cytoscape.js CSS rules are all set up so you can start modelling immediately.

The Palette (✦) tab in the sidebar shows what's available. Node types include Class, Instance, Concept, and Ontology. Edge types include rdfs:subClassOf, rdf:type, owl:ObjectProperty, and several SKOS relations.

Hover any palette item and click ✎ to rename it or change its colour, or click × to remove it. Use + Add node type / + Add edge type at the bottom to extend the palette with your own types.

💡 The palette is saved inside the .boxes file, so every collaborator who opens your file gets the same types automatically.
Boxes — OWL template loaded, Palette tab open
2

Add the ontology header

OWL best practice is to start with an owl:Ontology node that declares the ontology's IRI. In the Palette, click Ontology to make it the active type, then double-click the canvas to place it.

With the node selected, open the Properties (☰) tab and set:

  • label — a human-readable name (e.g. Animals)
  • @id — the ontology IRI (e.g. ex: or http://example.org/animals#)
💡 Press Esc after placing the node to deactivate the Ontology type, so the next double-click doesn't accidentally place another one.
Boxes — owl:Ontology header placed
3

Add classes

Place the owl:Class nodes that form the vocabulary of your ontology. In the Palette, click Class, then double-click the canvas once for each class. Select each node and set its label and @id in the Properties tab.

For the Animals ontology, place:

  • LivingThing — root of the hierarchy
  • Animal
  • Mammal and Bird
  • Dog, Cat, and Eagle
💡 Don't worry about positions yet — the Layout step will arrange everything automatically. All edits in the Properties tab are live; the canvas updates as you type.
Boxes — class nodes placed (no edges yet)
4

Build the class hierarchy

Connect your classes with rdfs:subClassOf edges to express the is-a relationship. In the Palette, click the are edge type.

To draw an edge:

  1. Hover over the source (more specific) class — a blue ● handle appears
  2. Drag from the handle to the target (more general) class and release

Draw these relationships:

  • Animal rdfs:subClassOf LivingThing
  • Mammal and Bird rdfs:subClassOf Animal
  • Dog and Cat rdfs:subClassOf Mammal
  • Eagle rdfs:subClassOf Bird
💡 In OWL, the convention is subclass (source) → superclass (target). The edge label "are" reads as "Dogs are Mammals".
Boxes — subclass hierarchy connected
5

Add instances

Named individuals are specific things that belong to one or more classes. In OWL, class membership is expressed via rdf:type.

In the Palette, click Instance, place a node for each individual, and set its @id in Properties. Then select the a edge type (rdf:type) and draw edges from each individual to its class.

  • Rex → rdf:type → Dog
  • Fido → rdf:type → Dog
  • Tweety → rdf:type → Bird
💡 The dotted purple rdf:type arrows are visually distinct from the grey rdfs:subClassOf arrows — this makes it easy to tell classes from individuals at a glance. Both colours are defined in the Stylesheet (✏) tab.
Boxes — individuals typed via rdf:type
6

Add object properties

Object properties describe typed relationships between resources. Add a Person class first, then use the ObjectProperty edge type to draw a hasOwner property from Animal to Person.

Selecting the new edge in the Properties tab shows:

  • label — hasOwner
  • @type — owl:ObjectProperty
  • rdfs:domain — set to the source class IRI automatically
  • rdfs:range — set to the target class IRI automatically
💡 ObjectProperty edges are rendered as dashed blue arrows to distinguish them from subclass arrowheads. You can customise colours and line styles in the Stylesheet (✏) tab.
Boxes — owl:ObjectProperty added
7

Run auto-layout

Open the Layout (⊕) tab and click Dagre. Dagre arranges directed graphs into a clean top-down hierarchy — ideal for a class tree.

Other layouts worth trying:

  • KLay — hierarchical with precise edge routing
  • Cola — physics-based; good for densely connected graphs
  • Concentric — radial; important nodes near the centre
  • Grid / Circle — geometric; useful as a starting point
💡 Select a subset of nodes with Shift+drag before running the layout to move only those, leaving the rest in place. Press F to fit all nodes on screen. Then hit Ctrl+S / ⌘S — layout positions are saved in the .boxes file.
Boxes — complete ontology with layout applied
8

Export to Turtle

Click Export → RDF / Turtle in the editor toolbar. Boxes serialises your graph using the namespace prefixes in the Context (@) tab, compressing long IRIs into readable CURIEs.

The resulting Turtle file can be opened directly in:

  • Protégé — the OWL ontology editor
  • Any SPARQL triple-store (Apache Jena, GraphDB, Stardog, …)
  • OWL reasoners (HermiT, ELK, Pellet)
  • Python via RDFLib

Here is the Turtle output for the Animals ontology:

turtle animals.ttl — generated by Boxes → Export → RDF / Turtle 
@prefix owl:  <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix rdf:  <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix ex:   <http://example.org/animals#> .

# Ontology declaration
ex: a owl:Ontology .

# Class hierarchy
ex:LivingThing a owl:Class .

ex:Animal a owl:Class ;
    rdfs:subClassOf ex:LivingThing .

ex:Mammal a owl:Class ;
    rdfs:subClassOf ex:Animal .

ex:Bird a owl:Class ;
    rdfs:subClassOf ex:Animal .

ex:Dog a owl:Class ;
    rdfs:subClassOf ex:Mammal .

ex:Cat a owl:Class ;
    rdfs:subClassOf ex:Mammal .

ex:Eagle a owl:Class ;
    rdfs:subClassOf ex:Bird .

ex:Person a owl:Class .

# Object property
ex:hasOwner a owl:ObjectProperty ;
    rdfs:domain ex:Animal ;
    rdfs:range  ex:Person  .

# Individuals
ex:Rex    a ex:Dog  .
ex:Fido   a ex:Dog  .
ex:Tweety a ex:Bird .
💡 You can also export to JSON-LD or RDF/XML. To import an existing Turtle file, use Import → RDF / Turtle — it merges into the current graph, preserving your palette and stylesheet.
