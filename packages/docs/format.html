📄 Format specification

The .boxes file format

A .boxes file is a UTF-8 encoded JSON document that captures a complete Labeled Property Graph: its nodes, edges, visual stylesheet, last-used layout, and optional JSON-LD context. Any tool that can produce this JSON can be read directly by the Boxes editor and by Cytoscape.js.

Introduction

The format is a thin envelope around two Cytoscape.js concepts — the elements JSON format and the stylesheet format — with a small amount of Boxes-specific metadata. Because both fields map directly to Cytoscape.js constructor options, any valid .boxes file can be rendered with no transformation: 

const { elements, userStylesheet, lastLayout } = JSON.parse(boxesFileContents);

cytoscape({
  container: document.getElementById('graph'),
  elements,
  style:  userStylesheet,
  layout: lastLayout ?? { name: 'preset' }
});

Top-level structure

FieldTypeDescription
elements object required Graph nodes and edges in Cytoscape.js elements format.
userStylesheet array optional Cytoscape.js stylesheet rules. Also accepted as stylesheet (legacy alias).
lastLayout object optional Name and options of the last-applied layout algorithm.
context object optional JSON-LD prefix context used when exporting to RDF formats.
version string optional Format version. Current value: "1.0.0". 
{
  "elements":       { "nodes": [...], "edges": [...] },
  "userStylesheet": [{ "selector": "...", "style": {} }, ...],
  "lastLayout":     { "name": "dagre", "options": {} },
  "context":        { "owl": "http://www.w3.org/2002/07/owl#" },
  "version":        "1.0.0"
}

elements

An object with two array fields. Both follow the standard Cytoscape.js elements JSON format. Either or both arrays may be empty.

FieldTypeDescription
nodesarrayArray of node objects.
edgesarrayArray of edge objects.

Node objects

Each node is a JSON object. Only data.id is strictly required; all other fields are optional.

FieldTypeDescription
data.id string required Unique identifier for the node. Must be unique across all nodes.
data.label string optional Human-readable display label. Referenced by the stylesheet via data(label).
data.* any optional Arbitrary additional properties (e.g. nodeType, year, @type). Addressable in stylesheet selectors.
position.x number optional Horizontal position in Cytoscape's model coordinate space.
position.y number optional Vertical position. If both x and y are 0 for all nodes, the editor auto-applies the saved layout.
classes string optional Space-separated CSS class names applied to the element. Usable in stylesheet selectors as .className. 
{
  "data": {
    "id": "alice",
    "label": "Alice",
    "nodeType": "person",
    "department": "Engineering"
  },
  "position": { "x": 120, "y": 80 },
  "classes": "highlighted"
}

Edge objects

Edges have no position. Both source and target must reference existing node IDs.

FieldTypeDescription
data.id string required Unique identifier for the edge.
data.source string required ID of the source node.
data.target string required ID of the target node.
data.label string optional Display label for the edge (relationship type).
data.* any optional Arbitrary additional properties.
classes string optional Space-separated CSS class names. 
{
  "data": {
    "id": "e-alice-bob",
    "source": "alice",
    "target": "bob",
    "label": "KNOWS",
    "since": "2019"
  }
}

userStylesheet

An array of stylesheet rules. Each rule is an object with a selector string and a style object. Rules are applied in order; later rules override earlier ones for the same property on the same element. 

[
  { "selector": "node",              "style": { "background-color": "#4a7fd4" } },
  { "selector": "node[nodeType = 'person']", "style": { "shape": "ellipse" } },
  { "selector": "edge",              "style": { "line-color": "#aaa" } }
]
Alias: The field name stylesheet is accepted as a legacy alias for userStylesheet when reading files. Always write userStylesheet in newly produced files.

Selector syntax

Selectors follow the Cytoscape.js selector specification, which is a subset of CSS selectors extended with data attribute matching.

SelectorMatches
nodeAll nodes
edgeAll edges
#aliceElement with id = "alice"
.highlightedElements with class highlighted
node[label]Nodes where label property exists
node[nodeType = 'person']Nodes where nodeType === "person"
node[nodeType != 'person']Nodes where nodeType !== "person"
node[weight > 10]Nodes where weight > 10 (numeric)
node[label *= 'Ali']Nodes where label contains "Ali"
node[label ^= 'Al']Nodes where label starts with "Al"
node:selectedSelected nodes
node.person, node.companyComma-separated list
Special characters in property names: If a data property name contains characters that are meaningful in CSS (e.g. @type), escape the special character with a backslash in the JSON string: "selector": "node[\\@type = 'owl:Class']". After JSON parsing the value becomes node[\@type = 'owl:Class'], which Cytoscape.js handles correctly.

lastLayout

Records which layout algorithm was last applied and with what options. When loading a file, the Boxes editor re-runs the layout only if all node positions are { x: 0, y: 0 }; otherwise saved positions are used.

FieldTypeDescription
namestringLayout algorithm name (see below).
optionsobjectAlgorithm-specific options (e.g. rankdir for dagre).

Built-in layout names

NameDescription
presetRestore node positions as stored. Use this when nodes have explicit position values.
gridRegular grid. Options: rows, cols.
circleNodes arranged in a circle.
concentricConcentric circles by a node metric.
breadthfirstHierarchical BFS tree. Options: directed, root.
coseForce-directed (built-in). Options: idealEdgeLength, nodeRepulsion.
randomRandom placement.
dagreDAG / hierarchy (extension). Options: rankdir ("LR"/"TB"), ranksep, nodesep.
colaConstraint-based force-directed (extension).
klayLayer-based directed layout (extension).
fcoseFast compound spring embedder (extension).
 
// Honour saved positions (most common for authored graphs)
"lastLayout": { "name": "preset" }

// DAG, left-to-right
"lastLayout": { "name": "dagre", "options": { "rankdir": "LR", "ranksep": 60 } }

// Circle, no options needed
"lastLayout": { "name": "circle" }
Tip: When generating a .boxes file programmatically without known positions, set lastLayout to any automatic layout (e.g. { "name": "dagre" }) and leave all node positions at { x:0, y:0 }. The editor will auto-apply the layout on load.

context

An optional JSON-LD context object that maps namespace prefixes to URIs. It is used exclusively by the Boxes RDF export pipeline (Turtle, JSON-LD, RDF/XML); Cytoscape.js and the embedding snippet ignore it entirely. 

"context": {
  "owl":  "http://www.w3.org/2002/07/owl#",
  "rdfs": "http://www.w3.org/2000/01/rdf-schema#",
  "skos": "http://www.w3.org/2004/02/skos/core#",
  "ex":   "https://example.org/"
}

Omit this field entirely (or pass {}) if the graph is not intended for RDF workflows.

version

A semver string identifying the format revision. Currently "1.0.0". The Boxes editor reads files with or without this field; tool authors should write it for forward-compatibility. 

"version": "1.0.0"

For tool authors

Generating a .boxes file without positions

If your tool doesn't compute layout, omit position on nodes (or set all positions to { x: 0, y: 0 }) and choose an automatic layout in lastLayout. The Boxes editor detects the all-zero case and runs the layout on import. 

// Tool output — positions unknown, let the editor lay it out
{
  "version": "1.0.0",
  "elements": {
    "nodes": [
      { "data": { "id": "a", "label": "Alpha" } },
      { "data": { "id": "b", "label": "Beta"  } }
    ],
    "edges": [
      { "data": { "id": "e1", "source": "a", "target": "b", "label": "LINKS_TO" } }
    ]
  },
  "userStylesheet": [
    { "selector": "node", "style": { "label": "data(label)", "background-color": "#4a7fd4" } },
    { "selector": "edge", "style": { "label": "data(label)", "line-color": "#aaa",
                                     "target-arrow-shape": "triangle", "curve-style": "bezier" } }
  ],
  "lastLayout": { "name": "dagre", "options": { "rankdir": "LR" } }
}

Minimal valid file

Only elements is required. All other fields have sensible defaults.

{
  "elements": {
    "nodes": [
      { "data": { "id": "n1", "label": "Hello" } },
      { "data": { "id": "n2", "label": "World" } }
    ],
    "edges": [
      { "data": { "id": "e1", "source": "n1", "target": "n2" } }
    ]
  }
}

ID rules

  • Node IDs and edge IDs share the same namespace — they must all be unique across both arrays.
  • IDs may contain any characters except whitespace. URL-safe alphanumeric strings are recommended.
  • IDs are case-sensitive: "Alice" and "alice" are different nodes.

Stylesheet ordering

Rules are applied in array order. Put broad selectors (e.g. node, edge) early and specific selectors (e.g. node[nodeType = 'person']) later so that specific rules correctly override general ones.

Reading .boxes files in other tools

Because the format is plain JSON you can load it with any JSON parser. The stylesheet uses the same property names as the Cytoscape.js style API, so you can cross-reference that documentation for the full set of supported properties and their allowed values.

Producing files programmatically: Call editor.exportGraph() from the Boxes API to get a serialisable object, then JSON.stringify(graph, null, 2) to write it. To read it back: editor.importGraph(JSON.parse(text)).
