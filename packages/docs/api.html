BoxesEditor

The main graph editor class. Wraps a Cytoscape.js instance and provides a full editing experience including node/edge manipulation, CSS styling, layout algorithms, undo/redo, and clipboard support. 

import { BoxesEditor } from 'boxes-core';
constructor new BoxesEditor(container, options?)

Creates a new graph editor instance attached to a DOM element.

ParameterTypeDescription
containerHTMLElementDOM element that will host the Cytoscape canvas. Required.
options.elementsobjectInitial graph data in Cytoscape elements format { nodes, edges }.
options.stylearrayInitial stylesheet rules array [{ selector, style }].
options.nodeTypesarrayNode type definitions. Each type: { id, label, data, color, shape, borderColor }.
options.edgeTypesarrayEdge type definitions. Each type: { id, label, data, color, lineStyle }.
options.contextobjectJSON-LD context for RDF export (prefix mappings).
options.layoutobjectInitial layout options. Defaults to { name: 'preset' }.
options.edgeHandlebooleanSet to false to disable interactive edge drawing. Default: true.
 
const editor = new BoxesEditor(document.getElementById('graph'), {
  elements: {
    nodes: [{ data: { id: 'n1', label: 'Alice' } }],
    edges: []
  },
  nodeTypes: [{ id: 'person', label: 'Person', color: '#4a7fd4' }],
  edgeTypes: [{ id: 'knows', label: 'knows', color: '#666' }],
  layout: { name: 'dagre' }
});

Node & Edge Operations

method addNode(data, position?, style?) → string

Adds a new node to the graph. Returns the new node's ID.

ParameterTypeDescription
dataobjectNode data. May include id (auto-generated if omitted), label, and custom properties.
position{ x, y } | nullPosition in graph coordinates. Defaults to a random position.
styleobjectInline Cytoscape style overrides for this node.
 
const id = editor.addNode(
  { label: 'Bob', age: 30 },
  { x: 200, y: 150 }
);
method addEdge(sourceId, targetId, data?, style?) → string

Adds an edge between two nodes. Returns the new edge's ID.

ParameterTypeDescription
sourceIdstringID of the source node.
targetIdstringID of the target node.
dataobjectEdge data. May include id, label, and custom properties.
styleobjectInline Cytoscape style overrides for this edge.
 
editor.addEdge('n1', 'n2', { label: 'knows' });
method removeElement(elementId)

Removes a node or edge by ID. Removing a node also removes its connected edges.

method updateElement(elementId, data, style?)

Updates the data (and optionally style) of a node or edge. Merges with existing data.

ParameterTypeDescription
elementIdstringID of the node or edge to update.
dataobjectData fields to merge. Existing fields not present here are preserved.
styleobject | nullStyle overrides to apply. Pass null to leave style unchanged.
method updateElementStyle(elementId, style)

Updates only the Cytoscape style of an element without touching its data.

Layout

method runLayout(layoutOptions)

Runs a layout algorithm on the current graph.

ParameterTypeDescription
layoutOptions.namestringLayout algorithm name. One of: random, grid, circle, concentric, breadthfirst, cose, dagre, cola, klay, euler, fcose, elk, spread, avsdf, cise.
layoutOptions.*anyAdditional layout-specific parameters (padding, spacing, etc.). See getLayoutParams().
 
editor.runLayout({ name: 'dagre', rankDir: 'LR', nodeSep: 50 });
method getAvailableLayouts() → string[]

Returns names of all registered layout algorithms.

method getLayoutParams(name) → object[]

Returns parameter definitions for a named layout, including labels, types, defaults, and constraints.

method getLastLayout() → object

Returns the most recently applied layout's name and options.

Graph I/O

method exportGraph() → object

Exports the complete graph state as a plain object. The returned object includes elements (Cytoscape JSON), style (stylesheet rules), nodeTypes, edgeTypes, context, and lastLayout. 

const graph = editor.exportGraph();
const json = JSON.stringify(graph, null, 2);
// Save or send graph...
method importGraph(graphData)

Loads a graph from a previously exported object. Replaces all current graph state including elements, stylesheet, types, and layout settings.

method getElements() → object

Returns the current graph elements in Cytoscape JSON format { nodes, edges }.

method loadElements(elements)

Replaces graph elements without affecting stylesheet or type configuration.

method exportPdf(options?) → Promise<Blob>

Exports the current graph as a PDF Blob. The cytoscape-pdf-export plugin is loaded lazily on the first call so it does not affect initial page load time.

OptionDefaultDescription
fulltrueExport the entire graph, not just the current viewport.
paperSize'LETTER'Paper size: LETTER, LEGAL, A4, A3, A2, A1, A0, TABLOID, or CUSTOM.
orientation'LANDSCAPE'PORTRAIT or LANDSCAPE.
margin52Page margin in PostScript points (72 pt = 1 inch).
bgfalseBackground colour (any CSS colour string), or false for transparent.
 
const blob = await editor.exportPdf({ paperSize: 'A4', orientation: 'PORTRAIT' });
const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url; a.download = 'graph.pdf'; a.click();
URL.revokeObjectURL(url);

Stylesheet

Boxes uses a user-editable stylesheet array on top of Cytoscape.js's built-in styling. Stylesheet rules are applied via Cytoscape selectors and are included in export data.

method getStylesheet() → array

Returns the current stylesheet rules as an array of { selector, style } objects.

method setStylesheet(rules)

Replaces the entire user stylesheet with the provided rules array.

editor.setStylesheet([
  { selector: 'node', style: { 'background-color': '#4a7fd4', 'color': '#fff' } },
  { selector: 'node[label="Person"]', style: { 'shape': 'ellipse' } },
  { selector: 'edge', style: { 'line-color': '#999' } }
]);
method addStyleRule(selector, style?)

Appends a new rule to the stylesheet.

method updateStyleRule(index, selector, style)

Replaces the rule at the given index.

method removeStyleRule(index)

Removes the rule at the given index.

Selection & Clipboard

method getSelected() → object

Returns the selected elements as { nodes, edges }.

method selectElements(elementIds)

Programmatically selects elements by their IDs. Deselects all others.

method removeSelected()

Removes all currently selected nodes and edges.

method copy()

Copies selected nodes (and their connecting edges) to the internal clipboard.

method cut()

Cuts (copies then removes) selected elements.

method paste()

Pastes clipboard contents, applying a position offset to avoid overlap with the originals.

History (Undo/Redo)

method undo()

Undoes the last graph-changing action. Supports up to 50 levels.

method redo()

Re-applies the most recently undone action.

Keyboard shortcuts: Ctrl+Z (undo), Ctrl+Y or Ctrl+Shift+Z (redo) are handled automatically when the editor container is focused.

Node & Edge Types

method getNodeTypes() → array

Returns the configured node type definitions.

method getEdgeTypes() → array

Returns the configured edge type definitions.

method addNodeOfType(typeId, position?) → string

Creates a new node using the default data and style of the given node type. Returns the new ID.

method setEdgeType(typeId)

Sets the active edge type used when drawing new edges interactively.

method getEdgeType() → object | null

Returns the currently active edge type definition.

Events

method on(eventName, handler)

Registers a handler for an editor event.

method off(eventName, handler)

Removes a previously registered event handler.

Event Reference

  • changeFired after any graph modification. Payload: { elements, style, ... } (same shape as exportGraph()).
  • selectFired when the selection changes. Payload: { nodes, edges }.
  • node:addFired when a node is added. Payload: { id, data, position }.
  • edge:addFired when an edge is added. Payload: { id, source, target, data }.
  • element:removeFired when elements are removed. Payload: { elements }.
editor.on('change', (graph) => {
  // Persist graph to server
  fetch('/api/graphs/1', {
    method: 'PUT',
    body: JSON.stringify(graph)
  });
});

Advanced

method getCytoscape() → cytoscape.Core

Returns the underlying Cytoscape.js instance for direct access to the full Cytoscape API — useful for advanced queries, custom animations, or Cytoscape extension usage.

Note: Direct mutations via the Cytoscape instance bypass Boxes' undo/redo system and event emission.
method destroy()

Destroys the editor instance, detaches all event listeners, and removes all DOM elements created by Boxes. Call this in component unmount hooks to prevent memory leaks. 

// Vue 3
onUnmounted(() => editor.destroy());

// React
useEffect(() => {
  return () => editor.destroy();
}, []);

defaultTemplates

Pre-configured graph templates that include node types, edge types, stylesheets, and JSON-LD context mappings. 

import { defaultTemplates } from 'boxes-core';
property defaultTemplates → object

An object whose keys are template IDs and values are template configuration objects.

Available Templates

IDNameDescription
blankBlankEmpty graph with minimal defaults.
owl-ontologyOntologyFull OWL/SKOS/SHACL template with node types for owl:Class, owl:Ontology, skos:Concept, and typed edge relationships. Includes JSON-LD context for common ontology prefixes.
arrowsArrowsTemplate mimicking the Arrows.app style for simple LPG diagrams.
 
const editor = new BoxesEditor(container, {
  ...defaultTemplates['owl-ontology'],
  layout: { name: 'dagre' }
});

RDF / Turtle I/O

Import and export graphs as RDF/Turtle (N3 format).

import { exportToTurtle, importFromTurtle, rdfExporter, rdfImporter } from 'boxes-core';
function exportToTurtle(graphData) → Promise<string>

Serialises a graph export object as an RDF/Turtle string. Uses the context field for prefix mappings.

const turtle = await exportToTurtle(editor.exportGraph());
// "@prefix owl: <...> .\n:Alice a :Person ..."
function importFromTurtle(turtleString, context?) → Promise<object>

Parses a Turtle string and returns a graph data object suitable for editor.importGraph().

property rdfExporter · rdfImporter

I/O plugin objects for use with the web app's io-manager. Implement { id, label, export/import }.

Converts a graph export to a JSON-LD document with @context, @graph, and typed nodes.

function importFromJsonLD(jsonldDoc, context?) → Promise<object>

Parses a JSON-LD object and returns a graph data object.

function graphDataToJsonLD(graphData) → object

Synchronous version that converts graph data to a JSON-LD object (no compaction step).

Arrows.app I/O

Import and export graphs in Arrows.app JSON format for round-trip compatibility with the popular LPG diagramming tool. 

import { importFromArrows, exportToArrows, arrowsImporter, arrowsExporter } from 'boxes-core';
function exportToArrows(graphData) → object

Converts a Boxes graph export to Arrows.app JSON format ({ nodes, relationships, style }). Preserves node positions, labels, properties, and Cytoscape visual styles. 

import { exportToArrows } from 'boxes-core';
const arrowsJson = exportToArrows(editor.exportGraph());
const text = JSON.stringify(arrowsJson, null, 2);
function importFromArrows(arrowsData) → object

Converts an Arrows.app JSON object to a Boxes graph data object suitable for editor.importGraph(). Translates Arrows style properties into Cytoscape stylesheet rules. 

import { importFromArrows } from 'boxes-core';
const response = await fetch('diagram.json');
const arrowsData = await response.json();
editor.importGraph(importFromArrows(arrowsData));
property arrowsExporter · arrowsImporter

I/O plugin descriptors for the web app's io-manager. Implement { id, label, export/import } and can be registered with registerExporter / registerImporter.

PDF Export

Export the graph as a PDF document using the cytoscape-pdf-export plugin. The plugin is loaded lazily on the first call so it does not contribute to initial bundle weight.

PDF export is available directly on the BoxesEditor instance via editor.exportPdf(options). In the web app, it is also exposed through the Export → PDF menu option provided by the built-in io-manager integration.

Note: The PDF plugin bundles PDFKit with embedded font data (~600 kB gzipped). Vite/Rollup code-splits it into a separate async chunk that is only fetched when the user first triggers a PDF export. 
// Standalone usage
const blob = await editor.exportPdf({
  paperSize: 'A4',
  orientation: 'PORTRAIT',
  bg: '#ffffff'
});

// Trigger browser download
const url = URL.createObjectURL(blob);
const a = document.createElement('a');
a.href = url; a.download = 'graph.pdf'; a.click();
URL.revokeObjectURL(url);

Options

OptionDefaultDescription
fulltrueFit the entire graph on the page rather than the current viewport.
paperSize'LETTER'LETTER, LEGAL, A4, A3, A2, A1, A0, TABLOID, or CUSTOM.
orientation'LANDSCAPE'PORTRAIT or LANDSCAPE.
margin52Page margin in PostScript points (72 pt = 1 inch).
bgfalseBackground colour string, or false for transparent.

Vue 3 Component

A Vue 3 single-file component wrapping the core BoxesEditor. Initializes Cytoscape in onMounted and exposes the editor instance via defineExpose. 

import { BoxesEditor } from 'boxes-vue';

Props

PropTypeDescription
elementsobjectInitial Cytoscape elements { nodes, edges }.
stylearrayInitial stylesheet rules.
nodeTypesarrayNode type definitions.
edgeTypesarrayEdge type definitions.
contextobjectJSON-LD context for RDF export.
layoutobjectLayout options.
widthstringCSS width of the container. Default: '100%'.
heightstringCSS height of the container. Default: '500px'.

Events

EventPayloadDescription
@changegraph objectEmitted after any graph change.
@select{ nodes, edges }Emitted on selection change.

Exposed (ref access)

Access the BoxesEditor instance directly via a template ref:

<BoxesEditor ref="editorRef" :node-types="types" />

// In script:
const editorRef = ref(null);
// Access: editorRef.value.editor  (the BoxesEditor instance)

React Component

A React 18 component using forwardRef + useImperativeHandle. Manages the Cytoscape lifecycle with useEffect. 

import { BoxesEditor } from 'boxes-react';

Props

PropTypeDescription
elementsobjectInitial Cytoscape elements { nodes, edges }.
stylearrayInitial stylesheet rules.
nodeTypesarrayNode type definitions.
edgeTypesarrayEdge type definitions.
contextobjectJSON-LD context for RDF export.
layoutobjectLayout options.
widthstringCSS width of the container. Default: '100%'.
heightstringCSS height of the container. Default: '500px'.
onChangefunctionCallback fired after any graph change. Receives the full graph object.
onSelectfunctionCallback fired on selection change. Receives { nodes, edges }.

Imperative handle (via ref)

Use React.forwardRef to access the underlying BoxesEditor instance:

const editorRef = useRef(null);

<BoxesEditor
  ref={editorRef}
  nodeTypes={nodeTypes}
  onChange={(graph) => console.log(graph)}
/>

// Access the BoxesEditor instance:
editorRef.current.editor.exportGraph();
