What you'll build

This tutorial walks you through building a simple Animals ontology in OWL 2. By the end you will have:

  • An ontology header node (owl:Ontology)
  • A class hierarchy — LivingThing → Animal → Mammal/Bird → Dog/Cat/Eagle
  • Individual instances (Rex, Fido, Tweety) typed via rdf:type
  • An object property (ex:hasOwner) linking animals to people
  • A valid Turtle / RDF export ready for Protégé or a SPARQL endpoint

The interactive demos below are live editors — you can click, zoom, and explore each stage. Use the Palette (✦) tab in each demo to see the node and edge types available.

1

Start from the template

Open Boxes and you'll see the template picker. Click "Ontology or RDF File" to start with the OWL palette and stylesheet pre-configured — you won't need to define node types or styles from scratch.

The palette (sidebar, ✦ tab) shows the node types and edge types that come with the template:

  • Node types: Class, Instance, Concept, Ontology
  • Edge types: rdfs:subClassOf, rdf:type, object/datatype properties, SKOS relations

You can customise these at any time — hover a type to reveal the ✎ edit and × delete buttons.

💡 Tip: If you already have a .boxes file saved from a previous session, click Open in the toolbar to reload it — the palette and stylesheet are restored automatically.
Boxes — OWL template loaded (empty canvas)
2

Add the ontology header

Best practice in OWL is to start with an owl:Ontology node that declares the ontology's IRI. In the Palette tab, click Ontology to make it the active type, then double-click the canvas to place it.

With the node selected, switch to the Properties (☰) tab and set:

  • label — a human-readable name (e.g. Animals)
  • @id — the ontology IRI (e.g. ex: or http://example.org/animals#)
💡 Tip: Press Esc after placing the node to deactivate the Ontology type, so the next double-click doesn't accidentally place another one.
Boxes — owl:Ontology header placed
3

Add classes

Now place the owl:Class nodes that form the vocabulary of your ontology. In the Palette, click Class, then double-click the canvas once for each class you want to add. Give each a label and an @id IRI in the Properties tab.

For the Animals ontology, place these classes:

  • LivingThing — root of the hierarchy
  • Animal
  • Mammal
  • Bird
  • Dog and Cat
  • Eagle

Don't worry about positions yet — the layout step will tidy everything up automatically.

💡 Tip: You can rename a node quickly by selecting it and editing the label field in the Properties tab. The canvas updates as you type.
Boxes — class nodes placed (no edges yet)
4

Build the class hierarchy

Connect your classes using rdfs:subClassOf edges to express the is-a relationship. In the Palette, click the are edge type (which maps to rdfs:subClassOf).

To draw an edge:

  1. Hover over the source (more specific) class — a blue ● handle appears
  2. Drag from the handle to the target (more general) class

Draw the following relationships:

  • Animal rdfs:subClassOf LivingThing
  • Mammal rdfs:subClassOf Animal
  • Bird rdfs:subClassOf Animal
  • Dog and Cat rdfs:subClassOf Mammal
  • Eagle rdfs:subClassOf Bird
💡 Tip: In OWL, the convention is that the subclass is the source node and the superclass is the target. The label "are" reads as "Dogs are Mammals".
Boxes — subclass hierarchy connected
5

Add instances

Ontologies often include named individuals — specific things that belong to one or more classes. In OWL, membership is expressed via rdf:type.

In the Palette, click Instance, place a node for each individual, and set its @id in Properties. Then select the a edge type (which maps to rdf:type) and draw edges from each individual to its class.

Add three individuals:

  • Rex — rdf:type → Dog
  • Fido — rdf:type → Dog
  • Tweety — rdf:type → Bird
💡 Tip: The dotted purple rdf:type edges are visually distinct from the grey rdfs:subClassOf arrows — this makes it easy to tell classes from individuals at a glance.
Boxes — individuals typed via rdf:type
6

Add object properties

Object properties describe relationships between individuals (or between classes, as domain/range constraints). Add a Person class first, then use the ObjectProperty edge type to draw a hasOwner property from Animal to Person.

When you select the new edge, the Properties tab shows:

  • label — hasOwner (the property name)
  • @type — owl:ObjectProperty
  • rdfs:domain — automatically set to the source class IRI
  • rdfs:range — automatically set to the target class IRI
💡 Tip: Object properties are rendered as dashed blue arrows to distinguish them from subclass arrowheads. You can customise colours and line styles in the Stylesheet (✏) tab.
Boxes — owl:ObjectProperty added
7

Run auto-layout

Once all nodes and edges are in place, open the Layout (⊕) tab and click Dagre. Dagre arranges directed graphs into a clean top-down hierarchy — perfect for a class tree.

Other layouts to try:

  • KLay — hierarchical with precise edge routing
  • Cola — physics-based; good for densely connected graphs
  • Concentric — radial, with important nodes near the centre

You can also lay out a subset of nodes: select them with Shift+drag before running the layout, and only the selected nodes move.

💡 Tip: After layout, press F to fit all nodes on screen. Then use Ctrl+S / ⌘S to save — the layout positions are stored in the .boxes file.
Boxes — complete ontology after Dagre layout
8

Export to Turtle

Click Export → RDF / Turtle in the toolbar. Boxes serialises your graph using the namespace prefixes in the Context (@) tab, compressing long IRIs into readable CURIEs.

The resulting Turtle file is valid OWL 2 and can be opened directly in:

  • Protégé — the OWL ontology editor
  • Any SPARQL triple-store (Apache Jena, GraphDB, Stardog, …)
  • OWL reasoners (HermiT, ELK, Pellet)
  • Python via RDFLib

Here is the Turtle output for the Animals ontology we've just built:

turtle animals.ttl — generated by Boxes Export → RDF / Turtle 
@prefix owl:  <http://www.w3.org/2002/07/owl#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix rdf:  <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix ex:   <http://example.org/animals#> .

# Ontology declaration
ex: a owl:Ontology .

# Class hierarchy
ex:LivingThing a owl:Class .

ex:Animal a owl:Class ;
    rdfs:subClassOf ex:LivingThing .

ex:Mammal a owl:Class ;
    rdfs:subClassOf ex:Animal .

ex:Bird a owl:Class ;
    rdfs:subClassOf ex:Animal .

ex:Dog a owl:Class ;
    rdfs:subClassOf ex:Mammal .

ex:Cat a owl:Class ;
    rdfs:subClassOf ex:Mammal .

ex:Eagle a owl:Class ;
    rdfs:subClassOf ex:Bird .

ex:Person a owl:Class .

# Object property
ex:hasOwner a owl:ObjectProperty ;
    rdfs:domain ex:Animal ;
    rdfs:range  ex:Person  .

# Individuals
ex:Rex    a ex:Dog  .
ex:Fido   a ex:Dog  .
ex:Tweety a ex:Bird .
💡 Tip: You can also export to JSON-LD or RDF/XML from the same menu. To import an existing Turtle or JSON-LD file into Boxes, use Import → RDF / Turtle — it merges into the current graph, preserving your palette and stylesheet.
